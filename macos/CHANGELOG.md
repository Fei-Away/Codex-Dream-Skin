# Changelog

## 1.5.2 — 2026-07-16

### 视觉

- `Miku Dream` 输入框改为白、淡青和淡粉的自然渐变，阴影同步改为主题色，不再出现突兀的黑色外围。
- 权限、计划、模型、添加内容和发送按钮增加清晰的文字、图标、边框、悬停与键盘焦点状态。

### 修复

- 移除浅色输入框上的大面积 `backdrop-filter`，避免 Electron 合成时出现深色边缘。
- 修正 composer 控件仍继承深色模式半透明白字、在浅色背景上难以辨认的问题。

## 1.5.1 — 2026-07-16

### 视觉

- 为 `Miku Dream` 增加配套侧边栏：粉青浅色底、低干扰星点纹理、青色图标与粉色选中点缀。
- 项目、任务和设置仍使用 Codex 原生控件，只调整背景、颜色、边框、悬停与键盘焦点状态。

### 改进

- renderer 会在根节点标记当前主题 ID，使专属组件样式不会影响其他主题包。

## 1.5.0 — 2026-07-16

### 新增

- 内置 `Miku Dream` 浅色主题包，使用纯净的粉青紫背景与右侧人物构图，不包含效果图中的假侧栏、按钮、卡片或输入框。
- 安装时会把项目内置主题同步到当前用户主题库；同名用户主题已存在时不会覆盖。
- 主题配置支持深色或浅色横幅文字，让明亮背景仍能保持原生标题和项目选择器可读。

### 改进

- 浅色壳的侧栏状态、建议卡图标和输入框装饰会跟随主题强调色，不再固定为红色。

### 修复

- 修正浅色主题任务页仍继承深色 Markdown 白字的问题，保持正文和顶部标题在明亮背景上清晰可读。
- 修正 JSON 状态查询把未探测的 CDP 和不同进程名的官方 Codex 误报为未运行。

### 安全

- 新主题仍通过 `127.0.0.1` CDP 注入，不修改官方 `.app`、`app.asar`、代码签名、API Key 或 Base URL。

## 1.4.0 — 2026-07-16

### 新增

- 自动监听已安装主题的 `theme.json`、背景图、样式和注入脚本变化，并在当前 Codex renderer 中重新加载最新主题。
- 外部编辑或 Theme Studio 更新主题后无需手动重启 Codex；更新期间文件暂时不完整时会等待下一次有效变更。

### 改进

- 主题热更新会先清理旧 renderer 状态，再注入新 payload，避免旧主题定时器和 MutationObserver 残留。

## 1.3.0 — 2026-07-16

### 新增

- 增加本地 Theme Studio 界面，可选择背景图、浅色/深色/跟随系统外观、主题文案和强调色。
- 增加实时 Codex 预览、自动加载状态和一键应用/恢复示例主题。

### 安全

- Theme Studio 只绑定 `127.0.0.1`，继续复用 Codex 已签名的 Node.js，不修改官方 `.app`、`app.asar`、API 配置或代码签名。

## 1.2.0 — 2026-07-16

### 新增

- 增加可开关的 macOS 自动加载：用户级 LaunchAgent 会用本机回环 CDP 预启动官方 Codex，并持续管理主题 injector。
- 安装后默认启用自动加载，新增启用、禁用和状态查询桌面入口。

### 改进

- Finder、Dock、终端等方式打开 Codex 时复用已带主题的实例；主动退出后会自动重新启动。
- 暂停、恢复和卸载会清理自动加载服务，避免恢复官方外观后被后台重新注入。

### 安全

- 自动加载仍只使用 `127.0.0.1` CDP，并验证官方 Codex 签名、Team ID、架构和 bundled Node；不修改 `.app`、`app.asar` 或代码签名。

### 修复

- 修正默认主题对不存在的时间戳背景图的引用，确保全新克隆后可以直接安装并启动。

## 1.1.1 — 2026-07-16

### 修复

- 不再用 `launchctl submit` 托管带调试口的 Codex：退出 SwiftBar / 关掉 Codex 后不应再被 launchd 自动拉起
- 暂停与完全恢复时清理 `com.openai.codex-dream-skin-studio.app` 作业

## 1.1.0 — 2026-07-16

### 新增

- SwiftBar 菜单栏入口（`Install Menu Bar.command`）：应用 / 暂停 / 换图 / 切换已保存主题 / 从图片文件夹加载 / 完全恢复
- 主题库（`themes/`）与图片投放目录（`images/`）动态加载，不再把 README 图库合成图当背景素材
- 按 Codex 应用浅色 / 深色自动切换皮肤壳（`data-dream-shell`）

### 改进

- CDP 已就绪时热切换主题（重启 injector + 短时注入），换图更快
- 注入校验放宽（项目选择器等可选），避免误杀已生效皮肤
- 注入守护优先 `nohup`；暂停状态与路径大小写下停止逻辑更稳
- 安装时不再强制 `appearanceTheme=dark`，只备份桌面外观相关键，便于恢复与自动适配

### 视觉

- 以原版暗色 portal CSS 为结构底，叠加 light 壳与更薄横幅遮罩，减轻「换图看不清」
- 示例纯横幅：`docs/images/banner-arina-hashimoto-pure-no-ui.png`（无人机 UI 合成）

### 说明

- `docs/images/gallery/` 仅为效果预览，不要当 `theme` 背景导入

## 1.0.0 — 2026-07-15

- 发布 macOS 通用主题制作器，而不是固定角色皮肤。
- 加入 Finder 选图、自动 JPEG 转换、主题命名和高级配色参数。
- 主页使用独立横幅，任务页使用背景与磨砂层，完整保留原生交互。
- 改为复用并验证 Codex 官方签名 Node.js，不再附带大型运行时或依赖全局 Node。
- 增加独立安装目录、桌面启动/定制/验证/恢复入口。
- 增加官方签名、CDP 端口归属、PID 身份、刷新重注入和真实 DOM 自检。
- 增加原子配置备份、精确恢复、静态测试、安装恢复循环和发布打包脚本。
- 清理固定角色内部命名；传送门主题仅作为可替换示例素材。
- 开源树：示例横幅改为无角色抽象几何图；验收截图不入库；补充 NOTICE / README 商标与安全边界说明。
