# 风险评估

## 一、安全风险评估

### 1.1 风险矩阵

| 风险项 | 概率 | 严重程度 | 风险等级 | 说明 |
|--------|------|---------|---------|------|
| CDP 被本机恶意程序利用 | 极低（<1%） | 中 | **低** | 仅绑定回环地址；攻击者需在用户机器上运行恶意程序 |
| Codex 更新导致注入失效 | 高（>50%） | 低 | **低** | 只需重新运行安装脚本 |
| 配置损坏 | 低（<5%） | 低 | **低** | 有完整备份和原子写入保护 |
| 托盘进程残留 | 低（<10%） | 极低 | **极低** | 恢复脚本自动关闭，也支持手动关闭 |
| 注入器守护进程崩溃 | 中（<20%） | 低 | **低** | watch 模式自动重新注入 |
| 图片导入失败 | 低（<5%） | 极低 | **极低** | 格式/大小校验在前，不写入损坏文件 |
| 杀毒软件误报 | 低（<5%） | 极低 | **极低** | 已修复快捷方式执行策略问题 |

### 1.2 风险缓解措施

| 风险项 | 缓解措施 | 责任人 |
|--------|---------|-------|
| CDP 被利用 | 仅绑定 `127.0.0.1`；BrowserId 验证；包身份验证；文档提示 | 用户 + 项目 |
| Codex 更新失效 | 自动发现最新 Store 包；不依赖硬编码路径 | 项目 |
| 配置损坏 | 原子写入；安装前备份；`-RecoverConfigBackup` 恢复 | 项目 |
| 托盘残留 | 命名 Mutex 防重复；恢复脚本自动关闭 | 项目 |
| 注入器崩溃 | watch 模式自动重连；状态文件记录 PID 可手动清理 | 项目 |
| 杀毒软件误报 | 快捷方式使用 `RemoteSigned` 而非 `Bypass` | 项目 |

## 二、功能风险评估

### 2.1 Codex DOM 结构变化

| 风险 | 说明 |
|------|------|
| **风险描述** | Codex 更新后 DOM 结构变化，皮肤选择器失效 |
| **概率** | 高（每次 Codex 大版本更新） |
| **影响** | 皮肤无效，但 Codex 本身功能不受影响 |
| **检测** | 验证脚本 (`verify-dream-skin.ps1`) 检查 DOM 标记 |
| **恢复** | 运行恢复脚本回到官方界面 |
| **修复** | 等待项目更新 `renderer-inject.js` 中的选择器 |

### 2.2 Node.js 版本兼容性

| 风险 | 说明 |
|------|------|
| **风险描述** | 未来 Node.js 版本变更可能不兼容 |
| **概率** | 低 |
| **影响** | 注入器无法启动 |
| **缓解** | 使用标准 Node.js API（`node:fs/promises`、`node:path`、`node:crypto`） |

### 2.3 端口冲突

| 风险 | 说明 |
|------|------|
| **风险描述** | 默认端口 9335 被其他程序占用 |
| **概率** | 中 |
| **缓解** | 自动扫描 100 个端口；可使用 `-Port` 自定义 |

### 2.4 多 Codex 版本同时运行

| 风险 | 说明 |
|------|------|
| **风险描述** | 多个 Codex 进程同时运行，皮肤无法正确注入 |
| **概率** | 低 |
| **检测** | 启动脚本检测多进程并抛出错误 |
| **恢复** | 关闭所有 Codex 后重新运行 |

## 三、运维风险评估

### 3.1 升级风险

| 风险 | 说明 |
|------|------|
| **风险描述** | 升级过程中断导致引擎文件不一致 |
| **概率** | 极低 |
| **缓解** | 原子替换（先复制到 staging 目录，SHA-256 校验，再 Move 替换） |
| **恢复** | 旧引擎备份到 `.engine-backup-*`，可手动恢复 |

### 3.2 磁盘空间

| 风险 | 说明 |
|------|------|
| **风险描述** | 频繁导入图片占用磁盘空间 |
| **概率** | 低 |
| **影响** | 图片存储在 `%LOCALAPPDATA%\CodexDreamSkin\images\` |
| **缓解** | 可手动清理不需要的图片 |

### 3.3 操作锁死锁

| 风险 | 说明 |
|------|------|
| **风险描述** | 并发操作因 Mutex 锁死 |
| **概率** | 极低 |
| **缓解** | Mutex 使用 `WaitOne(0)` 非阻塞模式；`AbandonedMutexException` 自动恢复 |

## 四、风险总结

### 4.1 可接受的风险

| 风险 | 理由 |
|------|------|
| CDP 回环暴露 | 需要同一用户运行恶意程序才能利用 |
| Codex 更新失效 | 可逆，不影响 Codex 正常使用 |
| 配置修改 | 有完整备份，可完美恢复 |
| 后台进程 | 可随时关闭，退出时自动清理 |

### 4.2 不可接受的风险（未发现）

| 风险 | 状态 |
|------|------|
| Token 泄露 | ✅ 未发现 |
| 远程代码执行 | ✅ 未发现 |
| 权限提升 | ✅ 未发现 |
| 数据外泄 | ✅ 未发现 |
| 永久性修改 | ✅ 未发现 |

### 4.3 风险评级：**低**

总体而言，Codex Dream Skin 的风险处于可接受的低水平。所有潜在风险都有明确的缓解措施或恢复方案。项目的安全设计和代码质量使风险被控制在合理范围内。
