# Codex Dream Skin 外部主题包 PRD

> 本文是外部主题包能力的产品权威。当前实现事实见 `docs/PROJECT.md` 与 `docs/platforms.md`；本文描述已确认目标，不表示功能已经发布。

## 1. 项目概述

- 背景：项目目前支持导入单张背景图，以及从平台自己的受管主题目录保存和切换主题，但没有供外部 AI 或开发者稳定使用的主题包格式。
- 要解决的问题：Kimi、Claude、其他 Agent 或人工开发者即使生成了背景、颜色和主题元数据，也缺少可验证、可分发、可直接导入的交付物。
- 目标用户：想安装第三方主题的 Dream Skin 用户；使用 AI 或代码工具制作主题的作者；维护主题契约与发行包的项目维护者。
- 期望结果：作者按公开契约生成一个 `.dreamskin` 文件，用户在 macOS 或 Windows 本地选择该文件，系统完成安全校验、摘要确认、原子安装，并可立即或稍后应用。

## 2. 用户与使用场景

| 用户/角色 | 典型场景 | 当前困难 | 期望结果 |
| --- | --- | --- | --- |
| Dream Skin 用户 | 从 Kimi 或第三方作者处拿到主题 | 只能手工复制图片和 JSON，容易放错路径或导入不安全内容 | 选择一个文件，看到来源与兼容性后安全导入 |
| AI / 主题开发者 | 根据用户描述设计并实现皮肤 | 不知道字段、资源限制、跨平台差异和验收方式 | 使用作者指南、Schema、示例和打包命令产出合格主题包 |
| 项目维护者 | 扩展主题字段或排查导入失败 | macOS 与 Windows 校验规则分散，错误语义不统一 | 以同一包协议、错误码和契约测试维护两端行为 |

## 3. 产品范围

### 本期包含

- 定义版本化、跨平台的 `.dreamskin` v1 本地主题包。
- 提供公开作者套件：格式说明、JSON Schema、示例源目录、Kimi/通用 Agent 提示词、校验与打包命令。
- macOS 菜单栏/命令行与 Windows 托盘/命令行均可选择本地 `.dreamskin` 文件。
- 导入前校验容器、清单、主题字段、资源哈希、图片类型/大小/尺寸、目标平台和最低 Dream Skin 版本。
- 导入前显示主题名、作者、包版本、目标平台、能力警告、可用预览和同 ID 冲突结果。
- 用户确认后原子安装到现有受管主题库；可选择“导入后立即应用”。
- 相同内容重复导入保持幂等；同 `packageId` 的不同内容或版本必须由用户确认替换。
- 安装与实时应用分别报告结果；Codex/CDP 暂不可用时，合法主题仍可安装并稍后应用。

### 本期不包含

- 不接入 Kimi、Claude 或其他模型 API；主题格式与具体 AI 解耦。
- 不支持 URL、在线市场、自动更新、远程仓库或后台下载。
- 不允许主题包携带或执行 JavaScript、Shell、PowerShell、可执行文件、动态库或任意 CSS。
- 不建立作者签名、证书、信誉、审核中心或商业分发体系。
- 不支持主题删除、跨设备同步、云备份或批量导入。
- 不承诺新增贴纸、任意 DOM、字体文件或新的渲染器布局能力；v1 只暴露已批准的声明式字段。

## 4. 核心功能

### 功能一：外部作者套件

- 用户目标：让 Kimi 或开发者无需理解项目内部安装路径，也能产出合格主题。
- 主要行为：读取公开契约与示例；在普通目录中准备 `manifest.json`、`theme.json` 和图片；运行校验/打包命令生成 `.dreamskin`。
- 用户可见结果：命令返回稳定的通过报告或字段级错误，不产生“看似打包成功、实际无法导入”的文件。
- 失败或限制：缺少必需文件、未知字段、不兼容能力、哈希不一致、图片超限或包含禁用文件时拒绝打包。

### 功能二：本地主题包导入

- 用户目标：从系统文件选择器选择一个 `.dreamskin` 并安全加入主题库。
- 主要行为：只读检查包文件；把允许条目流式写入隔离暂存区；验证并生成导入报告。
- 用户可见结果：校验通过后显示主题摘要；失败时显示稳定错误码、问题字段和不产生持久修改的说明。
- 失败或限制：不兼容当前平台、包格式过新、路径/条目异常、压缩炸弹风险、资源或 Schema 失败均阻断导入。

### 功能三：冲突处理与原子安装

- 用户目标：更新同一主题时不丢失当前可用版本。
- 主要行为：以 `packageId` 识别主题身份；相同内容直接报告已安装；不同内容要求用户取消或确认替换；替换通过备份与原子重命名提交。
- 用户可见结果：成功后主题出现在“已保存主题”；失败后旧主题仍可使用且暂存文件被清理。
- 失败或限制：显示名称相同但 `packageId` 不同的主题允许并存，界面必须同时展示作者/版本或短 ID 以便区分。

### 功能四：应用与结果反馈

- 用户目标：导入后直接使用，或先安装以后再切换。
- 主要行为：确认页提供“导入后立即应用”；安装成功后复用现有平台切换/注入链路。
- 用户可见结果：分别显示“已安装”“已应用”或“已安装，等待下次应用”。
- 失败或限制：若 active theme 尚未提交，任何失败都保留原主题；若主题已安全选中但 Codex/CDP 暂不可用，则不把运行时不可用误报为包损坏。

## 5. 约束与依赖

- `.dreamskin` v1 是单文件 ZIP 容器，但导入器不得把归档路径直接整包解压到文件系统。
- 容器只允许 Store/Deflate、UTF-8 文件名、唯一条目和明确白名单；禁止加密、多卷、符号链接、硬链接、绝对路径、路径穿越、控制字符和重复条目。
- 建议上限：包文件 32 MiB；背景 16 MiB；预览 4 MiB；`manifest.json` 与 `theme.json` 各 256 KiB；普通文件不超过 8 个；总解压后内容 24 MiB。
- 图片仍沿用现有安全限制：PNG/JPEG/WebP，单边不超过 16384px，总像素不超过 50MP。
- 主题包必须离线可验证、离线可安装，不上传主题、图片或用户路径。
- 必须继续遵守：只使用本机回环 CDP；不修改官方 `.app`、WindowsApps、`app.asar`、签名、API Key 或 Base URL。
- 当前上游 `main` 尚不包含 PR #123 的载荷字符串加固；接受外部自由文本前必须合入该修复或等价实现。
- v1 需要统一的便携字段和平台能力报告；平台暂不渲染的字段不得静默宣称生效。

## 6. 验收标准

| 场景 | 操作或触发 | 可观察结果 |
| --- | --- | --- |
| Kimi 作者流程 | 按提示词生成示例目录并运行校验/打包 | 生成一个可被两端导入的 `.dreamskin`，报告包 ID、版本、哈希和目标平台 |
| macOS 合法导入 | 从菜单或 CLI 选择官方示例包 | 显示摘要，确认后进入主题库，并可立即/稍后应用 |
| Windows 合法导入 | 从托盘或 CLI 选择同一示例包 | 得到与 macOS 一致的身份、版本、错误语义和主题结果 |
| 非法归档路径 | 包含 `../`、绝对路径、符号链接或重复条目 | 导入在持久写入前失败，主题库与当前主题字节不变 |
| 压缩炸弹/超限 | 条目或总解压大小超过限制 | 流式读取被中止，暂存清理，不分配或写入超出上限的数据 |
| Schema/哈希失败 | 修改主题字段或资源而不更新哈希 | 显示稳定错误码与字段/资源位置，不能确认安装 |
| 平台不兼容 | 包未声明当前平台或最低版本过高 | 显示兼容性原因，不进入安装阶段 |
| 重复导入 | 导入相同 `packageId`、版本和内容哈希 | 幂等返回“已安装”，不创建重复主题目录 |
| 更新冲突 | 同 `packageId` 但内容或版本不同 | 必须取消或确认替换；失败时旧主题可继续使用 |
| 应用环境不可用 | 安装成功但 Codex/CDP 不可用 | 明确区分“已安装”和“待应用”，主题仍能从已保存列表找到 |
| 禁止代码 | 包含 `.js`、`.css`、`.sh`、`.ps1`、可执行文件或动态库 | 无论是否被 manifest 引用都拒绝整个包 |

## 7. 待确认事项

- 导入确认页的图形预览在 macOS 菜单栏环境如何呈现：第一版可退化为系统预览 + 元数据确认，不能因此绕过确认门禁。
- macOS ZIP 条目读取采用系统工具流式输出还是引入固定、可审计的归档库，需要在实现前完成最小安全能力验证；不影响产品格式与业务流程。
- v1 便携主题字段的最终列表、平台能力警告和颜色映射由 `docs/THEME_PACKAGE.md` 与 JSON Schema 在首个实施模块中固化。
