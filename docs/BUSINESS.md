# Codex Dream Skin 外部主题包业务蓝图

> 本文从 `PRD.md` 推导角色、对象、流程、状态与规则。状态表和业务规则是语义事实源，图纸是派生视图。

## 1. 业务全景

- 核心价值如何产生：外部作者把声明式设计交付为标准主题包；本地导入器证明它兼容且不含可执行能力；用户确认后把它安全加入现有主题库。
- 业务从哪里开始、在哪里结束：从作者准备主题源目录开始，以用户得到“已安装/已应用”结果，或得到无持久副作用的拒绝/取消结果结束。
- 最重要的成功结果：同一个主题包可被 macOS 和 Windows 识别、验证、安装和追踪来源。
- 最重要的失败结果：任何不可信、超限、不兼容或冲突未确认的包都不得改变已安装主题和当前主题。

## 2. 角色与权限边界

| 角色 | 目标 | 可以做什么 | 不能做什么 | 数据范围 |
| --- | --- | --- | --- | --- |
| AI / 主题开发者 | 产出可分发主题 | 使用作者套件创建、校验、打包；声明作者、版本与素材信息 | 不能要求导入器执行代码、访问网络或写任意本地路径 | 自己创建的主题源目录与包文件 |
| 本地用户 | 安装并使用主题 | 选择本地包、查看摘要、确认/取消/替换、选择是否立即应用 | 未确认时不能发生持久安装；不能绕过兼容与安全门禁 | 自己的 Dream Skin 主题库与当前主题 |
| Dream Skin 导入器 | 执行可信转换 | 只读检查、不可信内容暂存、验证、生成报告、按用户决定原子安装 | 不能运行包内代码、整包信任解压、静默联网或自动覆盖 | 单次导入会话、受管暂存区、受管主题库 |
| 项目维护者 | 演进主题契约 | 发布 Schema、示例、能力矩阵和兼容规则 | 不能把未实现的平台字段宣传为可用 | 仓库契约、测试夹具和发行包 |

## 3. 核心业务对象

| 对象 | 含义 | 由谁产生/维护 | 与其他对象的关系 | 是否有生命周期 |
| --- | --- | --- | --- | --- |
| 主题源目录 | 作者工作区中的 manifest、theme 和资源 | AI / 主题开发者 | 打包后形成主题包 | 否 |
| 主题包 | `.dreamskin` 单文件、不可变的外部交付物 | 作者套件产生；导入器只读 | 产生一次导入会话，可安装为主题 | 否 |
| 导入会话 | 一次选择、校验、确认、安装的过程记录 | 导入器 | 引用候选包、报告和安装结果 | 是 |
| 导入报告 | 包身份、兼容性、警告、错误与内容哈希 | 共享验证核心 | 决定是否允许用户进入确认/安装 | 否 |
| 已安装主题 | 主题库中的规范化主题、资源和来源记录 | 导入器/平台主题库维护 | 可成为当前主题，可被同 ID 新版本替换 | 是 |
| 当前主题 | 平台 active store 选中的主题 | 现有切换链路维护 | 来自某个已安装主题或本地自定义主题 | 是 |

## 4. 核心业务流程

### 流程一：作者创建主题包

- 触发者与触发条件：AI 或开发者收到主题需求并取得作者套件。
- 正常过程：准备纯背景与声明式配置；运行本地校验；修复错误；打包为 `.dreamskin`；把包与必要的素材权利说明交给用户。
- 完成结果：包具有稳定 `packageId`、版本、目标平台、资源哈希与可复查的校验结果。
- 失败、禁止或人工处理：禁用文件、未知字段、素材超限、哈希或兼容信息失败时不得生成“合格”产物。

### 流程二：用户导入与应用主题包

- 触发者与触发条件：用户从 macOS 菜单/CLI 或 Windows 托盘/CLI 选择本地 `.dreamskin`。
- 正常过程：受控读取 → 暂存 → 完整校验 → 显示摘要 → 用户确认 → 原子安装 → 可选立即应用。
- 完成结果：主题进入已保存主题库，并明确报告安装与应用两个结果。
- 失败、禁止或人工处理：校验失败直接拒绝；同 ID 不同内容进入冲突决定；取消与失败均清理暂存并保留原主题。

## 5. 关键状态与转换

### 导入会话

| 对象 | 当前状态 | 触发动作/事件 | 目标状态 | 限制或副作用 |
| --- | --- | --- | --- | --- |
| 导入会话 | 已选择 | 开始检查本地包 | 校验中 | 只读外部文件；只允许写隔离暂存区 |
| 导入会话 | 校验中 | 所有容器、Schema、资源、兼容与 payload 门禁通过 | 待确认 | 生成导入报告；尚未写主题库 |
| 导入会话 | 校验中 | 任一门禁失败 | 已拒绝 | 清理暂存；主题库和当前主题不变 |
| 导入会话 | 待确认 | 用户取消 | 已取消 | 清理暂存；无持久副作用 |
| 导入会话 | 待确认 | 无冲突且用户确认 | 安装中 | 开始受管主题库事务 |
| 导入会话 | 待确认 | 同 `packageId` 内容或版本不同 | 身份冲突 | 必须等待用户取消或确认替换 |
| 导入会话 | 身份冲突 | 用户取消 | 已取消 | 旧主题不变 |
| 导入会话 | 身份冲突 | 用户确认替换 | 安装中 | 旧目录先备份，新目录发布后再清理备份 |
| 导入会话 | 安装中 | 原子发布成功 | 导入完成 | 主题已进入主题库；可进入应用动作 |
| 导入会话 | 安装中 | 发布失败且补偿完成 | 安装失败 | 旧主题仍可用；暂存与中间目录被清理 |

[查看经过渲染与验证的导入会话生命周期](./diagrams/theme-import-session-lifecycle.html)

### 已安装主题与当前主题

| 对象 | 当前状态 | 触发动作/事件 | 目标状态 | 限制或副作用 |
| --- | --- | --- | --- | --- |
| 已安装主题 | 不存在 | 合法包安装成功 | 已安装 | 以 `packageId` 为持久身份 |
| 已安装主题 | 已安装 | 用户选择应用 | 当前选中 | 写入平台 active store 前必须再次验证规范化主题 |
| 已安装主题 | 当前选中 | 用户切换其他主题 | 已安装 | 主题仍保留在已保存主题库 |
| 已安装主题 | 已安装/当前选中 | 同 ID 新版本确认替换 | 替换中 | 失败必须恢复旧目录；当前选中状态保持可解释 |
| 已安装主题 | 替换中 | 发布成功 | 已安装或当前选中 | 保持替换前是否当前选中的语义 |

## 6. 关键业务规则

- `packageId` 是安装身份，显示名称不是身份；显示名称允许重复，但必须可通过作者、版本或短 ID 区分。
- 同 `packageId`、同版本、同内容哈希的导入是幂等操作，不创建重复副本。
- 同 `packageId` 但内容或版本不同视为更新冲突，不能静默覆盖。
- 校验通过不等于已安装；只有用户明确确认后才能进入主题库事务。
- 主题包只表达数据与受控设计字段，不能获得代码执行、网络、Codex 配置、凭证或任意路径写入权限。
- 包内任何未声明或禁用条目都会拒绝整个包，不能“忽略危险文件后继续”。
- 安装结果与实时应用结果分开：运行时暂不可用不把合法包标为损坏；安装失败也不能被“稍后应用”掩盖。
- 单平台包可以使用统一格式，但必须声明目标平台；在非目标平台导入时明确拒绝。
- 素材权利由作者/分发者负责声明；导入器显示信息但不宣称完成法律审核。

## 7. 异常与边界场景

| 场景 | 系统或人工应如何处理 | 用户可见结果 |
| --- | --- | --- |
| 包在校验过程中被替换或修改 | 使用已打开文件描述符/稳定快照校验；检测变化即拒绝 | “包在读取期间发生变化”，无持久修改 |
| 包含未知、重复、加密或超限条目 | 停止读取并清理暂存 | 稳定错误码和具体条目 |
| 包声明支持当前平台但使用该平台不支持的能力 | 按契约决定阻断或明确降级警告；不得静默假装生效 | 确认页显示能力差异 |
| 已有相同内容 | 幂等结束，不重复安装 | “主题已安装”并提供应用入口 |
| 已有同 ID 不同内容 | 等待用户取消或确认替换 | 显示旧/新版本、作者和哈希摘要 |
| 替换中断或磁盘不足 | 恢复备份，保留旧主题 | “安装失败，旧主题已保留” |
| 安装成功但 Codex 未运行 | 保留已安装主题；按用户选择设置或不设置当前主题 | “已安装，等待应用” |
| 预览缺失或不可解码 | 背景资源可作为预览；仍无可用预览时显示元数据而非跳过确认 | “无预览图”，仍需确认 |
