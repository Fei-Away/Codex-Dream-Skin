# Manager Changelog

## 0.5.0 — 2026-07-18

### 新增

- schema v1 现支持完全自适应、部分语义色覆盖和完整十色色板，并统一支持 `appearance`、`art`、`palette.accent`、RGBA 与 alpha hex；保存和导出只保留用户实际声明的字段。
- 主题库动态发现 Manager、平台主题仓库和随包预设，以来源限定的选择键与内容指纹识别活动主题；同 ID 的不同来源不再互相覆盖。
- 平台引擎通过版本化 `runtime-capabilities.json` 声明固定命令、路径、Node 门槛和主题能力；Manager 在应用前给出兼容诊断，并可把只读或不兼容主题保存为显式降级的副本。

### 改进

- 主题编辑器在保存前按与后端一致的 RGBA 合成规则列出全部可读性问题，并提供需用户主动确认的一键修复；旧主题不再只显示英文校验错误。
- 受管引擎缓存改用资源内容指纹刷新；活动会话使用平台、进程启动时间、可执行路径、Browser ID 与主题目录共同校验，避免复用陈旧 PID 状态。
- 本地图片预览缓存改到应用数据目录并有界清理；图片单边上限统一为 16384 px，仍保持 50 MP、16 MB 与完整解码限制。
- 普通语义色继续只接受严格 Hex 和整数 `rgb()` / `rgba()`；`palette.accent` 额外保留受限的 `hsl()`、`oklch()` 与 `oklab()` 能力并独立上报兼容性。

### 安全

- 同版本 capability manifest 可忽略未知的根级 evidence 与 `themeFeatures` 标志，但未知 manifest 版本、paths、commands 和参数仍严格拒绝。
- 平台脚本、参数与资源路径均来自随包白名单，主题内容不能拼接命令、扩大 asset scope 或修改官方 Codex 安装。

## 0.4.0 — 2026-07-17

### 新增

- 主题编辑器覆盖 schema v1 的名称、品牌副标题、主题文案、项目前缀与按钮文案、状态、引言和可选推广元数据；内置主题与当前主题继续通过“保存为新主题”保护原件。
- 编辑器可选择本地 PNG / JPEG / WebP 替换主图、即时预览并提取自适应十色色板，也可一键采用新图配色或恢复原图。
- 增加 Codex-like 的 Home、Task、Settings 和 Narrow 四路交互模拟预览；侧栏、卡片、菜单、开关、输入区及 selected / focus / disabled 等状态可以点击检查，并始终标注为模拟预检。
- 十项颜色均提供色块、文本值和 alpha 滑杆联动，接受 `#RGB`、`#RGBA`、`#RRGGBB`、`#RRGGBBAA` 及受限 `rgb(...)` / `rgba(...)`。

### 改进

- 透明色验证使用稳定的白色画布 → 背景 → 面板 → 前景逐层合成，再检查实际表面的文字和强调色对比度。
- 状态探测改为单飞请求；只在窗口聚焦且页面可见时以六秒基准刷新，失败指数退避，状态未变化时不重绘编辑器。
- 替换主图或视觉字段后使旧实机预览失效，避免把已经过期的 `preview.*` 继续展示为当前效果。

## 0.3.0 — 2026-07-17

### 新增

- 增加本地主题编辑器，可编辑名称、文案和安全色板；内置或当前主题会另存为副本。

### 修复

- 明确区分组件可用、换肤环境已配置、会话运行中和热切换就绪；恢复官方外观不再误报“引擎未安装”。
- 状态每 1.8 秒静默刷新，并在窗口回焦时立即检测；单次后台探测失败不会清空已知状态。
- 主题已经注入时不再因当前路由缺少 Hero、卡片或输入框而误判失败并延迟回滚。
- 无实机截图时只展示“构图预检 · 非实机”，主题改动会使过期实机预览失效。

## 0.2.0 — 2026-07-16

### 新增

- 从图片创建主题时自动提取稳健的明暗调色板，并在导入与创建时强制校验正文/弱化文字至少 4.5:1、主要强调色至少 3:1 对比度。
- 主题卡片与详情页在没有实机截图时生成接近 Codex 结构的布局预览，并明确标注“布局预览”，不再把原始背景图冒充实机效果。
- 增加主题图片构图、安全区、明暗模式、可读性和验收规范，并在导入引导中直接展示关键要求。

### 修复

- Windows 应用主题改为有界热切换并等待注入验证；失败时同时回滚活动主题和当前页面，避免“界面已变但按钮一直转”或假成功。
- 移除隐藏的 PowerShell 重启弹窗，改为管理器内可见确认；过期 state 不再让主题误标为“使用中”。
- 通用主题使用真实 16:9 图片构图与简洁布局；浅色外观下侧栏、次要文本和控件现在保持可读。
- Windows 使用不透明无阴影窗口、深色内描边和 120 ms 尾随圆角更新，减少白边、黑框和拖拽缩放卡顿。

### 安全

- 图片解码增加边长和内存上限；主题仍保持纯声明式，不执行包内脚本、CSS 或远程资源。

## 0.1.0 — 2026-07-16

### 新增

- 增加 Windows / macOS 共用的 Tauri 2 管理器，以及环境检查、四步首次引导、主题库、搜索、预览、快速切换、诊断和恢复入口。
- 增加 schema 1 严格校验、图片格式与尺寸检查、展开大小限制、压缩包路径防护、同名导入安全改名和原子写入。
- 调用随应用打包的固定平台引擎；应用失败或取消重启时回滚活动主题。

### 修复

- 自定义标题栏窗口取消外围 8px 透明槽，并在 Windows 使用 DWM 圆角偏好与原生圆角区域裁剪，彻底去除四周黑色矩形边；最大化时自动恢复直角区域。
- 管理器实时验证本机 CDP 会话并区分“首次启用需重启一次”和“当前会话可立即热切换”，避免应用按钮与引导文案误导。
- Windows 与 macOS 发布包现在只携带各自平台的换肤引擎资源，修复 WiX 因 macOS 中文文件名导致 MSI 链接失败的问题。

- 首次引导使用不透明遮罩并停止在重复渲染时重播淡入动画，避免底层页面闪现。
- 规范化 Windows `\\?\` 路径并统一 PowerShell UTF-8 输出，避免本地图片预览失败和错误提示乱码。
- Windows 调试版也使用 GUI 子系统，启动管理器时不再闪出命令行窗口。
- 安装引擎前增加明确确认；授权后安全关闭 Codex，失败时尝试重新打开，且皮肤运行中禁止直接修复引擎。
- 修复图片创建 / 主题包导入已成功写盘却被前端误报失败，以及恢复后旧主题仍误标“使用中”的状态问题。

### 安全

- 主题包保持纯声明式，不加载包内脚本、CSS、远程图片或远程字体。
- 管理器启用受限 CSP 和本地图片 asset scope；主题应用、恢复与引擎安装只调用固定命令及固定脚本。
